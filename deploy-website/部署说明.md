## 🌐 支持的部署平台

### ✅ 最简单：直接上传静态文件

**任何支持静态网站托管的平台都可以：**

| 平台 | 方法 |
|------|------|
| **虚拟主机/云服务器** | 把整个文件夹上传到网站根目录 |
| **宝塔面板** | 上传到 `/www/wwwroot/你的域名/` |
| **Nginx/Apache** | 解压到网站根目录即可 |
| **OSS/COS 对象存储** | 开启静态网站托管功能后上传 |

## ⚙️ Nginx 配置示例

```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/deploy-website;
    index index.html;

    # SPA 路由支持 - 非常重要！
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 启用 Gzip 压缩
    gzip on;
    gzip_types text/css application/javascript image/svg+xml;
}
```

## ⚙️ Apache .htaccess 配置

```apache
RewriteEngine On

# SPA 路由回退
RewriteBase /
RewriteRule ^index\.html$ - [L]
RewriteCond %{REQUEST_FILENAME} !-f
RewriteCond %{REQUEST_FILENAME} !-d
RewriteRule . /index.html [L]
```

## 📋 注意事项

### ❗ 最重要：SPA 路由配置

**必须配置 URL 重写规则！** 否则直接访问子页面会 404。

所有路由都要回退到 `index.html`，由前端 JS 处理路由。

---

### 📍 子目录部署

如果部署到子目录（如 `https://domain.com/humanos/`），需要修改：

1. `index.html` 中所有 `/assets/` 改为 `./assets/`
2. 确保所有资源路径使用相对路径

---

## 📞 问题排查

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 页面空白 | JS 路径错误 | 检查浏览器控制台 404 错误 |
| 刷新 404 | 未配置 SPA 路由 | 添加 nginx/apache 重写规则 |
| 样式丢失 | CSS MIME 类型 | 服务器确保 `.css` 返回 `text/css` |
