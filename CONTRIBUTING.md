# 镜心 · 添薪（CONTRIBUTING）

> MindMirror · 镜心 —— 一面问己之镜

## 一 · 项目简介

「镜心」取「以心为镜，照见己身」之意。作答 48 题（12 维 × 4 题），映照一位
历史人物（东方文人 / 治国能臣 / 科技先贤 / 西方哲 / 西方科学共 5 域 150 位）。
项目不涉心理量表、不打分、不存敏感数据；纯以人文笔法作 12 维向量之心理
侧写，落点是「他者即镜」式的自我观照。

- 站名：**MindMirror**（主）/「镜心」（中文副名 / 产品名）
- 仓库：见 `package.json` 的 `homepage`
- 部署：GitHub Pages（`base: '/MindMirror/'`）
- 持久化：浏览器 `localStorage` 键 `mindmirror-v1`
- 资源：宣纸底纹 + 朱砂印章 + 水墨 SVG 肖像 + 霞鹜文楷 / 思源宋体子集

## 二 · 四条底线（不可越）

添内容、提 PR、提 Issue 之前，请先读这一节。**凡触底线者，PR 一律关闭。**

| # | 底线 | 含义 | 反例 |
|---|---|---|---|
| 1 | **不涉政治** | 不得以条目、文案、轶事、配图为某政权、政党、领袖、当代政治议题背书或贬抑 | 把维特根斯坦写成「反犹者」或「反法西斯斗士」 |
| 2 | **不涉民族宗教** | 不得就民族优劣、宗教正邪、文化高低下断；涉宗教者以学术史与思想史笔法出之 | 写耶稣会士时使用「异端」「野蛮」等词；将某民族整体「污名化」 |
| 3 | **不涉现代争议** | 性别 / 种族 / 阶级 / 性向 / 殖民 / 奴隶 / 大屠杀等议题，不替古人立「现代价值观」之判；写的是古人当时的选择与代价 | 给玄奘贴「文化挪用」标签；为康德之种族观写「批判声明」 |
| 4 | **不涉性别歧视** | 任何古人条目 / 文案 / 视觉都不应暗示某性别天然低劣；女性人物与男性人物享有同等条目深度与镜头比例 | 班昭只给一行小传；居里夫人称「居里之妻」 |

> 第 3 条尤其重要：古人有其时代局限，**展示**即可；**评价**请以「后世如何看」之
> 史料性语态出之，不可以今律古；亦不可以古非今。

## 三 · 提 PR 流程

1. **Fork** → 建分支 `<type>(<scope>): <subject>`，例：
   - `feat(east-statesman): 添张居正 / 范雎 / 李斯 3 人`
   - `fix(reflection): 雷达 320px 不溢出`
   - `docs(golden-cases): 扩列 18 例`

2. **域 / 任务** 大小划分：
   - **小**（单条文案 / 单题 / 单个修复）：≤ 100 行 diff
   - **中**（单人条目 / 单域 5 题 / 单组件）：100-500 行
   - **大**（单域 30 人 / 单域 30 张肖像 / 跨组件）：开 Issue 先议

3. **必跑**（提 PR 前）：
   ```bash
   npm run typecheck     # 0 error
   npm run test          # 单测全过
   npm run prettier      # 风格统一
   npm run build         # 构建通过
   node scripts/gen-portraits.mjs   # 肖像同步重生
   ```

4. **PR 描述模板**（必填）：
   - **What**：改了什么 / 添了什么
   - **Why**：为何改
   - **How**：技术要点（如有）
   - **Test**：跑了什么、贴图 / 截图
   - **Risk**：是否有破坏性 / 是否触及第 2 节底线

5. **Code review**：见第七节。

## 四 · 人物规范

### 4.1 选人原则

- **正史可考**：以二十五史 / 断代史 / 学术专著为底；戏曲 / 笔记 / 传说须显式标注「传说」。
- **有可识之行迹**：单纯「留名一句」者不收；须有可叙述之故事。
- **区段有别**：每域 30 人分布在 5 区段（极端外 / 偏外 / 中段 / 偏内 / 极端内），
  每区段 6 人。新增人物须明确归属区段（见 4.4）。

### 4.2 条目字数

| 字段 | 字数 | 备注 |
|---|---|---|
| `id` | 小写 + 域前缀 | 例 `el-laozi` / `es-shangyang` / `wp-socrates` |
| `name` | 1-3 字 | 中文名；西人用其最通行译名（见 §4.5） |
| `era` | 2-8 字 | 朝代 / 时期标签 |
| `signature` | 4-12 字 | 文言定场句，半文言半白话可 |
| `bio` | 30-80 字 | 白话小传；以正史为据 |
| `matchBlurb` | 含 `{{name}}` 占位 | 例「汝与 {{name}} 同，……」，须含一个具体意象 |
| `anecdotes[*].title` | 2-6 字 | 名曰「X 之 Y」 |
| `anecdotes[*].body` | 20-60 字 | 以正史 / 共知典为限；不溢美、不苛责 |
| `echoes` | 2-3 个 id | 同区段优先；跨段限 ≤ 1 |

### 4.3 不写

- 不写现代价值观判断（见底线 3）
- 不写"伟大 / 杰出 / 卓越"等空洞褒义形容词
- 不写"残忍 / 愚昧 / 反动"等苛责语；如需评其过失，写「后世论者以为……」
- 不杜撰对话（"据说他说过……"不算）
- 不写同代人对他的评价（"当时人称……"易杜撰）

### 4.4 区段向量参考（每域 30 人 = 5 区段 × 6 人）

| 区段 | 思辨 | 行动 | 革新 | 意志 | 备注 |
|---|---|---|---|---|---|
| A 极端外 | ≥ 0.85 | ≥ 0.85 | ≥ 0.90 | ≥ 0.80 | 突破 / 范式 / 敢赌 |
| B 偏外 | 0.65-0.85 | 0.70-0.85 | 0.65-0.85 | 0.80-0.90 | 守正 / 辅佐 / 中度风险 |
| C 中段 | 0.40-0.60 | 0.50-0.70 | 0.50-0.70 | 0.60-0.80 | 稳健 / 中庸 / 务实 |
| D 偏内 | 0.20-0.40 | 0.30-0.55 | 0.30-0.55 | 0.40-0.70 | 内省 / 处世 / 慎行 |
| E 极端内 | ≤ 0.20 | ≤ 0.30 | ≤ 0.30 | ≤ 0.40 | 静守 / 调和 / 全身 |

> 其他 6 维（情感 / 群我 / 审美 / 学识 / 处世 / 时间 / 风险 / 表达）由该人物行迹
> 推断，无固定阈值，但须与 5 区段气质一致。

### 4.5 西人译名

按《辞海》《不列颠百科全书》中文版 / 商务印书馆汉译名著通行译名出之；自创译名
须在 PR 描述说明。例：

| 通行名 | 不收的写法 |
|---|---|
| 柏拉图 | 柏拉童 / 普拉图 |
| 亚里士多德 | 亚理斯多德 / 亚里斯多德 |
| 维特根斯坦 | 维根斯坦 / 维特斯坦 |
| 萨特 | 沙特 / 萨特（惟）|
| 海德格尔 | 海德格 / 海德格（惟）|

### 4.6 已退休 / 拒收清单（不得再新增）

> 维护一份「争议人物 / 不收人物」清单，新人若要 PR 加入，须在 Issue 单独议。

- 当代政治人物（生年 > 1900 且仍具政治影响者）
- 重大争议未定的在世学者 / 在世作家
- 已严重神话 / 戏说化的虚构人物（《封神》《三国演义》原型须标「原型」，不可径直收录）

## 五 · 肖像规范

### 5.1 画布与版式

- **画布**：480 × 640（viewBox），无外框
- **三段式**：
  - **顶段**（高约 80px）：朝代小章（左上，篆体）+ 4 字题词（右上，朱砂）
  - **中段**（高约 440px）：著名场景（背景）+ 人物剪影（前景，姿态由 `buildScene` 渲染）
  - **底段**（高约 120px）：姓名（行书 / 楷书，居中）+ 朱砂方印（右下）+ 副标（小字，姓名下方）
- **滤镜**：从 `ink / ink-strong / wash / haze / paper` 五套中选 1-3 套叠加
  - `ink` 轻度水墨扰动（默认）
  - `ink-strong` 浓墨
  - `wash` 大面积渲染
  - `haze` 远山虚化
  - `paper` 宣纸底纹

### 5.2 朝代色板

8 张主色板（见 `scripts/portraits/common.mjs` 之 `PALETTES`）：

| key | 朝代 | 主色 | 备注 |
|---|---|---|---|
| `spring` | 春秋 | `#7a5a2e` | 老旧黄 |
| `warring` | 战国 | `#5e3a20` | 沉褐 |
| `westernHan` | 西汉 | `#6a5a45` | 灰赭 |
| `easternJin` | 东晋 | `#4e6a4a` | 冷绿 |
| `tang` | 唐 | `#b8842a` | 金黄 |
| `song` | 宋 | `#5068a0` | 冷蓝 |
| `yuan` | 元 | `#8a6a2a` | 土黄 |
| `ming` | 明 | `#a02828` | 朱红 |
| `qing` | 清 | `#3a4055` | 藏青 |

每色板含 `primary / secondary / ground / ink / seal / accent / label` 7 字段；
朱砂 `seal` 一律为 `#a82828` 区间。

> 西人（`west-*`）沿用所处世纪最贴近的色板（如 `modernEarly` 多取 `tang` 暖调；
> `modernLate` 多取 `ming` 朱调）。可在 `figures.west-*.mjs` 中重写 `palette`。

### 5.3 印章

- **位置**：底段右下角
- **尺寸**：60 × 60 px
- **内容**：3 或 4 个中文字
  - 3 字：竖排
  - 4 字：田字格（每格约 28 × 28 px）
- **字体**：篆体加粗（16-20px）
- **颜色**：朱砂 `seal`
- **内容规则**：
  - 优先取人物字号 / 著作名 / 关键典故
  - 避讳使用「皇」「帝」「神」等可能引起误读的极端字
  - 不写现代术语

### 5.4 题词（4 字朱砂）

- 位置：顶段右上
- 内容：取人物气质之一面（4 字），例：
  - 老子「紫气东来」
  - 庄子「庄周梦蝶」
  - 孔子「杏坛讲学」
  - 牛顿「万有引力」
- 避用「伟大 / 杰出 / 第一人」类形容

### 5.5 文件命名

- `<domain>/<id>.svg`，例：`portraits/east-literati/laozi.svg`
- 留待设计师替换的占位图也用此路径，覆盖前请先 PR 上 `Placeholder` 命名存档

## 六 · 12 维向量构建规则

> 12 维元信息见 `src/domain/traits/trait.dimensions.ts`。
> 5 区段阈值见 §4.4。

### 6.1 五维主干（权重 ≥ 1.0）

| 维 | lo（0）| hi（1）| 评估依据 |
|---|---|---|---|
| 思辨 | 务实 | 形而上 | 思辨著作 / 抽象程度 |
| 行动 | 沉思 | 果决 | 行迹 / 决断速度 |
| 革新 | 守成 | 突破 | 制度 / 思想 / 技术之破立 |
| 意志 | 随顺 | 刚毅 | 逆境中是否守其志 |
| 处世 | 进取 | 内省 | 退隐 vs 入世 |

### 6.2 七维辅（权重 ≤ 1.0）

| 维 | lo | hi | 评估依据 |
|---|---|---|---|
| 情感 | 克制 | 外露 | 诗文 / 言论 / 私生活之情感强度 |
| 群我 | 自我 | 群体 | 独行 vs 群体 / 门人 |
| 审美 | 理性 | 感性 | 诗文书画 / 美学主张 |
| 学识 | 通识 | 专精 | 著述之精深 / 广博 |
| 时间 | 当下 | 永恒 | 关注当下 vs 思至永恒 |
| 风险 | 求稳 | 敢赌 | 重大决策之赌性 |
| 表达 | 平实 | 华美 | 文风 / 论辩之繁简 |

### 6.3 评值流程

1. 读人物正文（25 史 / 学术专著）
2. 列出 12 维初值（0-1）
3. 与同区段邻居人物交叉对照（保证 5 区段各 6 人不重叠）
4. 由维护者集体过一遍；遇 ≥ 0.1 之差须说明
5. 更新 `figures.<domain>.ts` 并跑 `golden-cases.md` 自检

> 向量无标准答案，但须**自洽 + 区段分布 + 与金样例不冲突**。维护者在
> `golden-cases.md` 留 1 条覆盖此人物的样例。

## 七 · 48 题构建规则

> 题库元信息见 `src/domain/items/item.types.ts`。

### 7.1 分布

- 12 维 × 4 题 = 48 题
- 每题 6 选项；主测一维，副影响 1-2 维
- 主维度 `primary.delta` 范围 ±1.5；副维度 `secondary.delta` 范围 ±0.5

### 7.2 题型（4 类，混用）

| format | 占比 | 含义 |
|---|---|---|
| `situational` | ~50% | 情境二择 / 多择 |
| `ranking` | ~15% | 排序 |
| `imagery` | ~20% | 意象 / 比喻 |
| `aesthetic` | ~15% | 诗文 / 美学 |

### 7.3 题目与选项规范

- `prompt`：4-12 字文言短句
- `promptGloss`：10-30 字白话注解
- `text`（选项）：4-8 字文言
- `gloss`（选项）：10-30 字白话
- 6 选项必须覆盖该维度的两端 + 4 中间区段，**不允许 6 项同向**
- 顺序用「由浅入深」或「由冷到热」等隐序

### 7.4 新增 / 修订题目

1. 在 `items.<domain>.ts` 添加/修改
2. 更新 `golden-cases.md` 中相应样例（如影响向量）
3. 题号连号（`el-001` 至 `el-048`），删除题须留 stub 注释，避免新题号误
4. 题库只增不删（PR 删题须议）

## 八 · 风格指南

### 8.1 文体

- **半文言**：人名、著作、年号、地名用通行写法；
  句中可用「之」「而」「以」「其」「亦」「乃」等虚词助雅
- **不堆典**：每条 ≤ 3 处典故；引文须有把握出处
- **不口号**：避免「伟大 / 杰出 / 永垂不朽」
- **不评判**：写「其行实……」不写「其人实乃……」

### 8.2 印章 / 题词

- 印章 3-4 个中文字（见 §5.3）
- 题词 4 字（见 §5.4）
- 避用俗字与生僻字；通用字表内 3500 字

### 8.3 代码风格

- TypeScript：`strict: true`；优先 `readonly` / `as const`
- 缩进 2 空格；行宽 100
- `npm run prettier` 自动格式化
- 不引新依赖（见 §3.5）

### 8.4 资源规范

- 肖像 SVG 7-13 KB / 张
- 整体 `dist/` 资源 ≤ 200 KB gzip（CI 卡死）
- 字体子集化（见 C11 任务）
- 不可用人物真实照片（涉肖像权）

## 九 · 提交前自检

逐项过一遍，缺一不可：

- [ ] **底线**（§2）— 文字 / 视觉 / 上下文均不触 4 条底线
- [ ] **人物**（§4）— 区段 / 字数 / 不写 / 译名均合
- [ ] **肖像**（§5）— 480×640 / 三段式 / 朝代色板 / 印章 3-4 字 / 题词 4 字
- [ ] **向量**（§6）— 5 区段各 6 人，新人物不与旧人撞区段
- [ ] **题目**（§7）— 12 维 × 4 题，6 选项覆盖两端
- [ ] **风格**（§8）— 半文言 / 无口号 / 无评判
- [ ] **资源**（§8.4）— SVG ≤ 13 KB / 总量 ≤ 200 KB gzip
- [ ] **命令**（§3.3）— typecheck / test / prettier / build / gen-portraits 全过
- [ ] **金样例**（C16）— 新增人物须留 1 条金样例
- [ ] **PR 模板**（§3.4）— 5 项必填

## 十 · Code review 标准

> Reviewer 应在 3 工作日内首响；超 5 工作日 ping `@maintainers`。

| # | 项 | 关注点 |
|---|---|---|
| 1 | **底线** | §2 4 条是否触犯？ |
| 2 | **正确性** | TS 类型 / 算法 / 数据正确？ |
| 3 | **可读性** | 命名清晰 / 注释到位 / 段长合理？ |
| 4 | **可测性** | 新逻辑有单测？E2E 仍 46/46？ |
| 5 | **可访问性** | 键盘 / 屏幕阅读器 / 减动效 / 颜色对比？ |
| 6 | **性能** | dist 增量 < 5 KB？FCP / LCP 不退化？ |
| 7 | **资源** | SVG 体积 / 字体子集 / 图像懒载？ |
| 8 | **i18n** | 字典双行 / 不漏新 key？ |
| 9 | **金样例** | C16 增列 / 复跑？ |
| 10 | **一致** | 与已有命名 / 模块 / 风格保持一致？ |

### 10.1 阻断（merge 之前必修）

- 触底线
- typecheck / test / build 失败
- 资源 / 性能预算超限
- 引入新依赖未经议

### 10.2 建议（合并后下次改）

- 命名小瑕疵
- 注释可更准
- 边界 case 补单测

### 10.3 点赞（直接合）

- 修 typo / 调字号 / 优化注释
- 1-2 行 hotfix（无新依赖）
