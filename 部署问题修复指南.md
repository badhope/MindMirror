# 🚀 HumanOS 部署404问题完全修复指南

## ❌ 问题现象
访问网站子页面直接404，首页正常
- ✅ `https://yourdomain.com/` 正常
- ❌ `https://yourdomain.com/assessment` 404
- ❌ `https://yourdomain.com/dashboard` 404

---

## ✅ 最快解决方案（30秒修复）

### 方案1: Apache虚拟主机 (cPanel/宝塔/面板)

**把这个文件上传到服务器网站根目录：**
```
文件位置：网站根目录/.htaccess
```

`.htaccess` 内容已经在dist目录里了，直接上传即可！

---

### 方案2: Nginx服务器

**在你的nginx配置中添加这一行：**
```nginx
location / {
    try_files $uri $uri/ /index.html;    # ✅ 这行就是SPA路由魔法！
}
```

**宝塔面板操作步骤：**
1. 网站 → 设置 → 配置文件
2. 在 `location /` 区块中修改成上面的样子
3. 保存并重载配置

---

### 方案3: 如果上面都不行

直接把 `dist/404.html` 内容上传，很多主机会自动识别404.html文件，自动实现跳转。

---

## 📋 检查清单

上传dist目录时，确保这些文件也一起上传：
```
dist/
├── .htaccess          # Apache配置 ✨ 最重要！
├── index.html
├── 404.html           # 降级方案
├── _redirects         # Netlify/Vercel
├── _headers
└── assets/
```

## 💡 为什么会这样？

**技术原理解释：**

```
传统网站（PHP/WordPress）:
  /assessment → 服务器找 assessment/ 文件夹
  找到了 → 返回里面的index.html

React SPA单页应用:
  只有一个 index.html 文件！
  /assessment 是JS在浏览器里模拟的路由！
  服务器找不到这个文件夹 → 404
```

**所以需要告诉服务器：**
> "找不到文件没关系，都给我返回index.html，JS自己会处理！"

---

## 🎯 验证是否成功

1. 清理浏览器缓存（Ctrl+Shift+R）
2. 直接访问 https://你的域名/assessment
3. ✅ 正常显示页面 = 修复成功！
4. ❌ 还是404 = 服务器没读取到.htaccess，需要重启Apache或确认mod_rewrite开启

---

## 📞 还有问题？

1. 检查 `.htaccess` 文件名是不是正确（开头有个点！）
2. 检查Apache有没有开启 `mod_rewrite` 模块
3. 检查网站根目录有没有 `AllowOverride All` 权限
4. Nginx用户一定要改配置文件，.htaccess不生效！
