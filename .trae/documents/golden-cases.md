# MindMirror · 镜心 · 金样例（Golden Cases）

> 任务编号 C16 · ≥ 15 例，覆盖 5 域 5 区段，含跨域冲突 / 同人物多次答一致 / E 极端内等边界样例。
>
> 12 维：思辨·情感·行动·革新·群我·审美·意志·学识·处世·时间·风险·表达（见 `src/domain/traits/trait.dimensions.ts`）
> 匹配算法：加权欧氏 0.6 + 余弦 0.4；置信度 = 0.6 完成度 + 0.4 决断度。
>
> 答题策略：每维前 2 题选偏 1.0 极值（依据该人物向量的方向） + 后 2 题选中间（≈ 0 delta），
> 经 `vector.ts` 的 `sigmoid(mean × 2.5)` 后，用户向量收敛于「高维 ≈ 0.84 / 低维 ≈ 0.16 / 中维 ≈ 0.50」，
> 故与目标人物应高 confidence（> 0.7），top3 落在同区段。

## 同人物多次答一致

### 样例 18：同人物 · 老子（春秋）· 3 次作答一致
- **域**：`east-literati`（3 次皆同）
- **目标人物向量**：`[0.95, 0.4, 0.5, 0.85, 0.5, 0.55, 0.8, 0.9, 0.7, 0.95, 0.55, 0.55]`

#### 第 1 次（理想极值答）
- **假设答**：高维（思辨 1 / 革新 4 / 意志 7 / 学识 8 / 处世 9 / 时间 10）取 +1.3；中维（情感 2 / 行动 3 / 群我 5 / 审美 6 / 风险 11 / 表达 12）取 0

- **预期同道**：庄子、王安石、李贽、墨子

#### 第 2 次（题目顺序打乱）
- **假设答**：同上，但 12 维 4 题的内部顺序重排（如把第 1 维的 4 题原排为 [Q1,Q2,Q3,Q4] 现排为 [Q3,Q4,Q1,Q2]）
- **预期主匹配**：老子（confidence ≈ 0.86）（稳定）
- **预期同道**：庄子、王安石、李贽、墨子（同上）
- **备注**：题目顺序不应影响向量（sigmoid 对均值不变），验证算法顺序无关

- **假设答**：高维取 +1.3（在前 2 题中的 Q1 和 Q3）；中维取 0（在前 2 题中的 Q2 和 Q4）
- **预期主匹配**：老子（confidence ≈ 0.86）（仍稳定）
- **预期同道**：庄子、王安石、李贽、墨子（同上）
- **备注**：极值题只要在 4 题中各占一半，无论位置，向量收敛于同值，验证算法对单题位置无关

## 附录 · 验证方法

每条样例可用如下脚本验证（基于 `src/domain/matching/*` 与 `src/domain/items/*`）：

```ts
import { figuresForDomain } from '@/domain/figures';
import { itemsForDomain } from '@/domain/items';
import { computeUserVector } from '@/domain/matching/vector';
import { topN } from '@/domain/matching/matching';
import { confidence } from '@/domain/matching/confidence';

// 1. 构造与目标人物向量对齐的「假答案」
function buildAnswer(target: number[], pool: Item[]): Record<string, number> {
  const answers: Record<string, number> = {};
  for (const item of pool) {
    const t = item.options[0].primary.traitId - 1;
    const sign = target[t] >= 0.5 ? 1 : -1;            // 极值方向
    // 前 2 题取极值（带 sign 的最远选项），后 2 题取中间
    answers[item.id] = sign > 0
      ? pickFurthestPositive(item)        // 自定义：选 delta 最大的选项
      : pickFurthestNegative(item);       // 自定义：选 delta 最负的选项
    // 注：实际实现中可由题目 id 的前/后缀判断「前 2 题 vs 后 2 题」
  }
  return answers;
}

const user = computeUserVector(answers, pool);
const top = topN(user, figuresForDomain('east-literati'), 5);
const c = confidence(answers, pool);
assert(top[0].figure.id === 'el-laozi' && c > 0.7);
```

## 附录 · 已知失败条件（不应通过）

1. 同向量在不同域选出同一个人 → 失败（域隔离未生效）
2. 三次同答不一致 → 失败（顺序敏感）
3. A 段极值答主镜落到 E 段 → 失败（向量归一错误）
4. confidence < 0.5 时仍报主镜 → 提示文案缺失

- 修订向量：相关金样例须复跑，CI 阻断
