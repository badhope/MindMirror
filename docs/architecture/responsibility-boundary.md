# MindMirror 前后端职责边界定义

> **文档版本**: v1.0.0
> **创建日期**: 2026-05-21
> **目标**: 明确前后端职责边界，指导前后端分离架构设计

---

## 1. 概述

### 1.1 设计原则

- **职责分离**: 后端专注数据和业务逻辑，前端专注视图和交互
- **API 驱动**: 前后端通过明确定义的 API 接口通信
- **渐进迁移**: 保持向后兼容，支持离线 fallback
- **性能优先**: 最小化网络请求，本地缓存优化

## 2. 后端职责 (Python/FastAPI)

### 2.1 数据存储

| 职责 | 说明 | 实现 |
|:-----|:-----|:-----|

### 2.2 业务逻辑

#### 2.2.1 测评计算

| 职责 | 说明 | 优先级 |
|:-----|:-----|:-------|
| 测评计算引擎 | 统一调度所有计算器 | 高 |
| SBTI/MBTI 计算 | 16型人格计算 | 高 |
| OCEAN 大五计算 | 大五人格计算 | 高 |
| 其他专业测评计算 | EQ、依恋、抑郁等 | 中 |
| 趣味测评计算 | 政治立场、吃货等级等 | 低 |

**计算器架构**:

```python
# backend/app/services/calculators/base.py
from abc import ABC, abstractmethod
from typing import Dict, Any, List

class BaseCalculator(ABC):
    @abstractmethod
    def calculate(self, answers: List[Dict[str, Any]]) -> Dict[str, Any]:
        pass

# backend/app/services/calculator_factory.py
class CalculatorFactory:
    _calculators: Dict[str, Type[BaseCalculator]] = {
        "sbti-personality": SBTICalculator,
        "ocean-bigfive": OceanCalculator,
        # ...
    }
```

### 3.4 离线支持

#### 3.4.1 IndexedDB 本地缓存

| 职责 | 说明 | 实现 |
|:-----|:-----|:-----|
| 测评数据缓存 | 题目、选项 | idb library |
| 计算结果缓存 | 最近的结果 | IndexedDB |
| 用户进度缓存 | 未完成的测评 | localStorage |
| 离线数据访问 | 无网络时读取 | Cache Strategy |

**缓存策略**:

```typescript
// 前端缓存层架构
class LocalCache {
  // 1. 优先从 IndexedDB 读取
  // 2. 缓存最近访问的数据
  // 3. 离线时使用缓存数据
  // 4. 网络恢复后同步
}
```

#### 3.4.2 本地 Fallback

| 职责 | 说明 | 实现 |
|:-----|:-----|:-----|
| API 不可用检测 | 网络状态监控 | navigator.onLine |
| 降级到本地计算 | 离线时本地计算 | CalculatorEngine |
| 数据同步 | 网络恢复后上传 | Background Sync |
| 用户提示 | 离线/在线状态 | UI Indicators |

**Fallback 流程**:

```
用户提交答案
      │
      ▼
┌─────────────────┐
│ API 可用?        │
└────────┬────────┘
         │
    Yes  │  No
    ┌────┴────┐
    ▼         ▼
┌─────────┐ ┌─────────────────┐
│ 发送    │ │ 使用本地计算器   │
│ 到后端  │ │ 计算结果        │
└────┬────┘ └────────┬────────┘
     │               │
     ▼               ▼
┌─────────────────────────┐
│ 更新本地 IndexedDB      │
│ (如果需要)               │
└─────────────────────────┘
```

## 4. 边界交互规范

### 4.1 数据流定义

#### 4.1.1 测评提交流程

```
Frontend                          Backend
    │                                 │
    │  1. 用户完成答题                │
    │  ─────────────────────────────▶ │
    │                                 │
    │  2. 前端收集答案               │
    │     - 验证必填项               │
    │     - 格式化答案数据           │
    │                                 │
    │  3. POST /api/v1/assessments/{id}/submit
    │     { answers: [...] }          │
    │  ─────────────────────────────▶ │
    │                                 │
    │  4. 后端验证答案               │
    │     - 参数校验                 │
    │     - 业务规则验证             │
    │                                 │
    │  5. 后端计算结果               │
    │     - 调用对应计算器           │
    │     - 生成报告数据             │
    │                                 │
    │  6. 返回结果                   │
    │     { success, data: result }  │
    │  ◀───────────────────────────── │
    │                                 │
    │  7. 前端渲染结果               │
    │     - 雷达图                   │
    │     - 详细报告                 │
    │                                 │
```

#### 4.1.2 数据同步流程

```
Frontend                          Backend
    │                                 │
    │  1. 应用启动                   │
    │  ─────────────────────────────▶ │
    │                                 │
    │  2. 获取用户信息               │
    │     GET /api/v1/users/me       │

    │                                 │
    │  3. 检查本地缓存               │

    │     ◀───────▶  ───────▶       │
    │                                 │
```

### 4.2 错误处理规范

#### 4.2.1 前端错误处理

| 错误类型 | 处理方式 | 用户提示 |
|:---------|:---------|:---------|
| 网络错误 | 使用本地 fallback | "离线模式" |
| 400 错误 | 提示具体错误 | "请检查输入" |
| 401 错误 | 跳转登录页 | "请先登录" |
| 404 错误 | 显示空状态 | "内容不存在" |
| 500 错误 | 重试机制 | "服务器错误，请重试" |

#### 4.2.2 后端错误处理

| 错误码 | 说明 | 返回格式 |
|:-------|:-----|:---------|
| 400 | 请求参数错误 | `{ error: { code: 400, message: "...", details: {} } }` |
| 401 | 未授权 | `{ error: { code: 401, message: "Unauthorized" } }` |
| 403 | 权限不足 | `{ error: { code: 403, message: "Forbidden" } }` |
| 404 | 资源不存在 | `{ error: { code: 404, message: "Not Found" } }` |
| 422 | 验证错误 | `{ error: { code: 422, message: "Validation Error", details: {} } }` |
| 500 | 服务器错误 | `{ error: { code: 500, message: "Internal Server Error" } }` |

## 6. 迁移策略

### 6.1 渐进迁移原则

1. **保持向后兼容**: 现有功能不能受影响
2. **可选 API 调用**: API 不可用时使用本地 fallback
3. **数据双写**: 初期同时写本地和后端
4. **灰度发布**: 逐步增加后端依赖

### 6.2 迁移顺序

| 阶段 | 迁移内容 | 优先级 | 说明 |
|:-----|:---------|:-------|:-----|
| 1 | 测评计算 (核心) | 高 | 最重要的业务逻辑 |
| 2 | 用户认证 | 高 | 安全相关 |
| 3 | 测评数据 | 高 | 静态数据迁移 |
| 4 | 测评提交 | 高 | 核心功能 |
| 5 | 成就系统 | 中 | 辅助功能 |
| 6 | 心情记录 | 中 | 数据分析 |
| 7 | 训练记录 | 中 | 数据分析 |
| 8 | 知识库 | 低 | 可缓存 |

### 6.3 测试策略

| 测试类型 | 覆盖范围 | 自动化程度 |
|:---------|:---------|:-----------|
| 单元测试 | 计算器算法 | 高 |
| 集成测试 | API 端点 | 高 |
| E2E 测试 | 完整流程 | 中 |
| 性能测试 | API 响应时间 | 中 |
| 离线测试 | Fallback 机制 | 低 |
