## 🎯 迁移核心原则

### ✅ **单人模式优先**
- 先做好单人体验，PVP/多人联机 **以后再说**
- 所有功能围绕单用户优化
- 暂不考虑数据冲突、竞态条件

### ✅ **渐进式迁移，零宕机**
- 永远保留前端计算作为降级方案
- 双写验证：前后端并行计算，抽样对比
- 用户无感知切换，随时可以回滚

### ✅ **问题前置解决**
- 迁移前识别所有潜在问题
- 每个阶段都有完整的回滚方案
- 每一行代码都有对应的测试用例

## 🚨 **所有已知风险与预解决方案**

| 风险编号 | 风险描述 | 发生概率 | 影响程度 | 预解决方案 |
|----------|---------|----------|----------|-----------|
| **R001** | JS vs Python 浮点数精度差异导致哈希不一致 | 100% | 中 | 所有数值保留4位小数，哈希计算时归一化到字符串 |
| **R002** | Math.random() 实现不同，随机事件结果不同 | 100% | 高 | Phase 1-4 禁用随机事件，Phase 5用统一种子算法 |
| **R003** | Modifier 叠加顺序不同，结果有偏差 | 高 | 写专用测试用例，固定叠加顺序：先加后乘 |
| **R004** | Date 进位逻辑不一致 (30/31天) | 高 | 统一用30天每月，不搞复杂的日历 |
| **R005** | 后端重启，所有会话丢失 | 中 | Phase 7做持久化之前，提示用户后端正在升级 |
| **R006** | 网络延迟导致感觉比本地卡 | 中 | 乐观更新：先发指令，立即预渲染结果，后端确认后修正 |
| **R007** | 计算出错用户进度报废 | 极低 | 永远保留前端计算作为最后的安全网 |
| **R008** | 浏览器禁用了CORS | 低 | 检测到跨域失败自动切前端计算，不影响体验 |
